## About me

I build systems that turn messy technical problems into clear, testable workflows. My work runs from reachability-aware vulnerability triage and market microstructure to quantitative decision support and web products for small businesses.

The way I work is AI-first. I describe what I want, Claude writes the first pass, I test the behavior before I read the code, then I direct the changes. My bar for done is one thing: working in production, tested, and clean. Performance comes first, and every module gets its own file.

I care about explicit assumptions, reproducible demos, real test coverage, and knowing when to override an AI-generated first pass.

## How I build

- Ship an end-to-end vertical slice before expanding scope.
- Keep calculations deterministic and testable, separate from any AI-generated explanation.
- Write down the tradeoffs and the designs I rejected, not just the one I kept.
- Treat AI as a collaborator whose output still needs verification and judgment.
- One responsibility per file. No monoliths, no dependencies I do not need.
